# 航空订单 AI 智能体演示

一个基于 `Spring Boot + Spring AI Alibaba + Agent Framework + Vue 3` 的航空客服演示项目。

项目采用左右分栏页面：
- 左侧展示订单列表
- 右侧提供 AI 智能体对话窗口

当前版本重点实现了航空客服的核心对话闭环，包括知识库问答、退票办理、更改预订，以及 Agent 过程监控与流式输出。

## 功能概览

### 1. 知识库问答
- 支持基于本地知识库文件回答常见问题
- 可回答预订航班、退票规则、更改预订规则、手续费等咨询
- 知识来源文件：[`知识库.md`](./知识库.md)

### 2. 退票办理
- 先核验预订号和客户姓名
- 查询订单后说明退票规则、手续费和限制
- 在聊天框内展示“确认 / 取消”按钮
- 用户点击“确认”后再执行退票
- 退票成功后自动刷新订单列表

### 3. 更改预订
- 支持更改预订日期
- 先核验预订号和客户姓名
- 收集新的出发日期后说明改签规则和手续费
- 在聊天框内通过按钮确认后执行改签
- 改签成功后同步刷新订单状态

### 4. 流式输出
- 右侧聊天窗口支持流式展示大模型回复
- 同时支持把工具调用过程流式插入到对话中
- 可看到类似：
  - `调用工具：query_order(...)`
  - `工具结果：query_order，...`

### 5. Multi-Agent
- 顶层使用路由 Agent 分发任务
- 当前包含 3 个子 Agent：
  - `knowledge_base_agent`
  - `cancel_booking_agent`
  - `change_booking_agent`

### 6. Agent 过程监控
- 支持查看当前会话的执行轨迹
- 可观察：
  - 大模型调用次数
  - 工具调用次数
  - MemorySaver 读写次数
  - 每一轮运行的事件时间线

## 核心实现点

### 1. 工具调用驱动业务
当前业务不是前端直接调“退票接口”或“改签接口”，而是由 Agent 通过工具调用推进流程。

主要工具包括：
- `search_knowledge_base`
- `query_order`
- `get_cancel_policy`
- `cancel_order`
- `get_change_policy`
- `change_order`

### 2. 会话与记忆
- 聊天上下文保存在内存中
- 订单数据保存在内存中
- 通过 `MemorySaver` 保存 Agent 运行上下文

### 3. 人工确认
- 退票和改签都采用页面按钮确认
- 不再依赖用户手动输入“确认退票”“确认改签”
- 交互更稳定，也更适合演示

## 本地启动

### 1. 启动后端

```bash
cd backend
export AI_DASHSCOPE_API_KEY=你的Key
mvn spring-boot:run
```

默认端口：`8080`

### 2. 启动前端

```bash
cd frontend
corepack pnpm install
corepack pnpm dev
```

默认端口：`5173`

如果你使用 `npm`，也可以：

```bash
npm install
npm run dev
```

## 构建

### 后端

```bash
cd backend
```

### 前端


## 当前实现边界

当前项目是演示型实现，已明确不包含以下能力：
- 数据库持久化
- 登录鉴权
- 多用户隔离
- 复杂票务规则引擎
- 生产级审批流
- 文件上传和语音能力

## 后续可扩展方向

- 接入数据库持久化订单与会话
- 扩展更复杂的改签规则
- 引入向量检索或 RAG 能力升级知识库
- 增加更细粒度的审批与审计能力
- 支持更完整的流式事件时间线展示
